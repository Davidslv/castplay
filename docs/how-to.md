## Record a cast from a real terminal

Use [asciinema](https://asciinema.org) to record, which produces a v2 `.cast` file:

```sh
asciinema rec demo.cast
# …do your thing in the terminal, then exit or Ctrl-D
```

Then either point a player at the file (`data-cast="demo.cast"`, served over http) or paste its contents into an inline `<script type="text/cast">` block (works from `file://`). castplay reads the header + `"o"` (output) events and ignores the rest — see the [cast-format reference](cast-format.md).

Keep recordings **linear** (type a command, see output, repeat). castplay appends text; it doesn't emulate a cursor, so avoid `clear`, progress bars, and full-screen tools like `vim` or `top` (see [architecture → where this strains](architecture.md#where-this-design-would-strain)).

## Inline a cast vs fetch it from a URL

`data-cast` accepts either form:

```html
<!-- Inline: reads from the matching <script id>. Works from file://. Preferred. -->
<pre class="term" data-cast="#session-1"></pre>
<script type="text/cast" id="session-1">
  {"version":2,"width":80,"height":10}
  [0.3,"o","$ ls\r\n"]
</script>

<!-- URL: fetched over http(s). Does NOT work from file:// (browsers block fetch). -->
<pre class="term" data-cast="casts/session-1.cast"></pre>
```

**Rule of thumb:** inline for self-contained pages and slide decks; URL when the cast is large, shared between pages, or generated by a build.

## Add or change colours

Colour comes from ANSI-SGR escape sequences inside the cast text. In a `.cast`, the escape byte is the JSON escape `\u001b`, followed by `[`, the code(s), and `m`:

```
\u001b[32m   green text starts
\u001b[0m    reset to default
\u001b[1m    bold on
\u001b[1;36m  bold + cyan in one sequence
```

Supported codes: reset (`0`), bold (`1`), and the eight colours `32 90 37 92 91 33 36 97`. Full table in the [cast-format reference](cast-format.md#supported-ansi-sgr-codes).

## Theme every player at once

The colour map is public. Override it **before** the players start (i.e. before `castplay.js` runs, or by editing entries and calling `Castplay.init()` yourself):

```html
<script>
  // define the palette override first…
  window.addEventListener('DOMContentLoaded', function () {
    Castplay.palette[32] = '#3fb968'; // a greener success green
    Castplay.palette[36] = '#58a6ff'; // a bluer cyan
  });
</script>
<script src="castplay.js"></script>
```

Because `castplay.js` auto-inits on load, the cleanest approach for a full re-theme is to set `Castplay.palette` and then drive `init()` yourself — see [the parser/renderer recipe](#use-the-parser-or-renderer-on-their-own) and [add players after load](#add-players-after-page-load). For a one-page deck, editing the `palette` defaults at the top of your vendored `castplay.js` is perfectly fine too.

## Run several players on one page

Just add more `[data-cast]` elements — `init()` finds them all. Each plays independently when it scrolls into view:

```html
<pre class="term" data-cast="#one"></pre>
<pre class="term" data-cast="#two"></pre>
<script type="text/cast" id="one">
  …
</script>
<script type="text/cast" id="two">
  …
</script>
<script src="castplay.js"></script>
```

## Add players after page load

`castplay.js` scans once on load. If you inject `[data-cast]` elements later (a single-page app changing routes, content loaded via `fetch`), call `init` again, optionally scoped to the new subtree:

```js
const container = document.querySelector('#new-section');
Castplay.init(container); // wires up any [data-cast] inside #new-section
```

There is currently no teardown API; see [architecture → where this strains](architecture.md#where-this-design-would-strain).

## Use the parser or renderer on their own

The pure helpers are exported, so you can use them without the player — for example to pre-render a cast to HTML at build time, or in a Node script:

```js
const { parseCast, ansiToHtml } = require('castplay');
const fs = require('fs');

const cast = fs.readFileSync('demo.cast', 'utf8');
const events = parseCast(cast); // [ [t,"o",text], … ]
const fullText = events.map((e) => e[2]).join('');
const html = ansiToHtml(fullText); // escaped HTML with <span> colour runs
```

## Troubleshooting

| Symptom                                                   | Cause                                       | Fix                                                                                          |
| --------------------------------------------------------- | ------------------------------------------- | -------------------------------------------------------------------------------------------- |
| Blank, never animates; opened from disk; using a URL cast | Browsers block `fetch()` on `file://`       | Inline the cast, or serve the page over http(s)                                              |
| `(cast failed to load …)` shown in the element            | The URL 404'd or the origin blocked it      | Check the path; serve over http; or inline it                                                |
| `(cast not found: #id)`                                   | No element matches the `#id` in `data-cast` | Ensure a `<script type="text/cast" id="id">` exists with that id                             |
| `(cast failed to parse)`                                  | A line isn't valid JSON                     | Each event line must be valid JSON; the escape byte must be `\u001b`, not a raw control char |
| Colours print as literal `[32m`                           | Missing escape byte                         | Write the escape as `\u001b` in the cast JSON                                                |
| Plays immediately, not on scroll                          | Browser lacks `IntersectionObserver`        | Expected fallback — content still shows                                                      |
| Output piles up instead of updating in place              | Cast uses cursor moves / redraws            | castplay is for linear sessions; re-record without `clear`/progress bars                     |
